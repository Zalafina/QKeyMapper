# FloatingButton Tooltip — Manual QToolTip::showText Pattern

## Problem

Qt's built-in tooltip mechanism (`QEvent::ToolTip` event generated by internal timer after `QEvent::Enter`) does NOT fire on `FloatingButtonWidget` — a top-level QPushButton with:
- `Qt::Tool | Qt::FramelessWindowHint | Qt::WindowDoesNotAcceptFocus` window flags
- `WA_TranslucentBackground` | `WA_NoSystemBackground` | `WA_ShowWithoutActivating`
- Custom `setStyleSheet("QToolTip { ... }")` which installs `QStyleSheetStyle` proxy
- `WS_EX_NOACTIVATE | WS_EX_TOOLWINDOW` native styles

Verified through `DEBUG_LOGOUT_ON` log: `QEvent::Enter` fires correctly with correct `toolTip()` text, but `QEvent::ToolTip` never arrives.

## Root Cause

Exact Qt internal mechanism unclear — likely `QStyleSheetStyle` proxy on the QPushButton prevents the tooltip timer from starting, or the combination of `Qt::Tool` + `WA_TranslucentBackground` affects Qt's tooltip event delivery. Debugging Qt internals was inconclusive.

## Solution: Manual QToolTip::showText (proven pattern)

The `armFloatingButtonMoveState` function already uses `QToolTip::showText()` successfully to show move-hint tooltips. Apply the same pattern for the button's own tooltip.

### Implementation in enterEvent:
```cpp
void FloatingButtonWidget::enterEvent(QEnterEvent *event)
{
    animateHoverTo(1.0);
    QPushButton::enterEvent(event);

    if (!toolTip().isEmpty() && !m_MousePassThrough) {
        QTimer::singleShot(700, this, [this]() {
            if (underMouse() && !toolTip().isEmpty()) {
                QToolTip::showText(mapToGlobal(rect().center()),
                                   toolTip(), this, rect());
            }
        });
    }
}
```

### Cleanup in leaveEvent/hideEvent:
```cpp
void FloatingButtonWidget::leaveEvent(QEvent *event)
{
    QToolTip::hideText();
    animateHoverTo(0.0);
    QPushButton::leaveEvent(event);
}

void FloatingButtonWidget::hideEvent(QHideEvent *event)
{
    QToolTip::hideText();
    resetHoverState();
    QPushButton::hideEvent(event);
}
```

### Design points:
- 700ms delay mimics Qt default tooltip delay
- `underMouse()` guard prevents stale timer from showing tooltip after mouse left
- `!m_MousePassThrough` guard prevents tooltip in mouse-passthrough mode
- No need to cancel pending `singleShot` timers — the `underMouse()` guard filters them
- Does NOT interfere with `armFloatingButtonMoveState`/`armFloatingButtonSyncGroupMoveState` which use their own `QToolTip::showText()` calls

**Why:** Qt built-in tooltip timer not working on FloatingButtonWidget; `QToolTip::showText()` already proven in move-armed state. When to revisit: if Qt bug is identified and fixed in a future Qt version.
